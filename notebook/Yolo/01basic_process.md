# YOLO 含义

YOLO（You Only Look Once）是一种流行的物体检测和图像分割模型

Ultralytics 于 2023 年发布的 YOLOv8 引入了新功能和改进，以增强性能、灵活性和效率，并支持全方位的视觉 AI 任务

## YOLO/ai 大概的工作流程（mode）

- train：训练
- validation：验证
- predict：预测/推理
- export：导出部署

## YOLO 的功能（Task）

- detection：目标检测
    - 功能：识别图像中的物体，并用 边界框（bounding box） 标出，同时分类。
    - 输出：每个目标的类别 + 边界框坐标（x, y, w, h）。
    - 应用：行人检测、交通标志识别、车辆检测、安防监控等。
    - 特点：YOLO 是单阶段检测器，速度快，适合实时检测。
- instance segmentation：实例分割
    - 功能：在目标检测的基础上，不仅给出边界框，还能生成 每个目标的像素级分割掩码。
    - 输出：类别 + 边界框 + 分割掩码（mask）。
    - 应用：自动驾驶（分割行人、车道线）、医学图像（分割器官或肿瘤）、图像编辑。
    - 特点：比检测任务更精细，模型更复杂。
- pose/keypoints：姿态估计/关键点检测
    - 功能：识别人体（或动物）关键点的位置，例如关节、手指、面部关键点。
    - 输出：关键点坐标 + 可选连接关系（skeleton）。
    - 应用：动作识别、体育分析、虚拟现实、舞蹈/运动捕捉。
    - 特点：需要对每个关键点单独预测，通常输出热力图或坐标列表。
- oriented detection：定向/旋转目标检测
    - 功能：检测 旋转的目标，输出 旋转边界框（x, y, w, h, θ），适合非水平/竖直的目标。
    - 输出：类别 + 旋转边界框参数。
    - 应用：航空影像中的建筑物、船只检测，文本检测（文本可能旋转）。
    - 特点：比普通检测更复杂，需要预测角度。
- classification：分类
    - 功能：给整张图像或者选定区域打标签。
    - 输出：类别概率（通常是 softmax 输出）。
    - 应用：图像识别（猫/狗/车）、疾病诊断（X 光片分类）、产品分类。
    - 特点：最基础的任务，只输出类别，不需要边界框或关键点。

## YOLO 模型名称含义

#### 常用后缀

| 后缀    | 含义          | 特点                              |
|-------|-------------|---------------------------------|
| **n** | nano        | 极小模型，参数少，推理速度最快，适合嵌入式或实时推理，精度最低 |
| **s** | small       | 小型模型，速度快，精度一般，适合轻量级应用           |
| **m** | medium      | 中型模型，平衡速度和精度，适合常规 GPU 训练/部署     |
| **l** | large       | 大型模型，更高精度，推理较慢，适合高性能 GPU 或离线推理  |
| **x** | extra large | 超大模型，参数最多，精度最高，适合研究或追求极致效果      |

#### 模型类型后缀

| 后缀                     | 含义           | 输出        |
|------------------------|--------------|-----------|
| **.pt**                | PyTorch 模型文件 | 默认训练/部署文件 |
| **.onnx**              | ONNX 格式      | 跨框架推理     |
| **.engine** / **.trt** | TensorRT 引擎  | GPU 高速推理  |

#### 任务类型后缀

| 后缀            | 含义                    | 输出                |
|---------------|-----------------------|-------------------|
| **-det** / 没加 | detection 默认          | 输出边界框 + 分类        |
| **-seg**      | instance segmentation | 输出边界框 + 分类 + mask |
| **-pose**     | pose / keypoints      | 输出关键点坐标           |
| **-cls**      | classification        | 输出类别              |
| **-oriented** | oriented detection    | 输出旋转边界框           |
